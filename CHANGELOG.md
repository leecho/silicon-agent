# Changelog

本文件记录值得发布说明的用户可见变更。

## Unreleased

### Added
- 设置页新增「调用日志」（默认关闭）：开启后记录每次模型调用的完整请求 payload + 原始响应 + token/缓存/耗时/成功失败，覆盖主会话、子代理、标题/建议生成、上下文压缩、记忆整理等全部调用，供调试与审计。支持按模型/类型/状态/关键词筛选，点开看单次调用明细（输入 messages、输出文本/工具调用/思考、错误详情）。单条超 256KB 截断、最多保留 5000 条，与既有「用量分析」（token 聚合）正交。
- 智能体「自我演化」：开启后，伴随体会在攒够新经历时自我反思、提出对自己人格（SOUL）的改写**提案**，经你**批准**才生效；身份锚（IDENTITY）永不被自动改动。「智能体」页新增演化区——开关、SOUL 版本史（可回滚）、待批准提案（看改动摘要后批准/拒绝）。由常驻后台线程按「最快每周一次且攒够新记忆」触发（默认关）；触发与补跑均从持久状态现算，重启不丢。
- 智能体人设拆分为「身份锚（IDENTITY）+ 人格（SOUL）」：身份锚承载名字/角色/硬边界、只人工编辑；人格（原指令）可编辑、并为后续「自我演化」预留可安全写回的层。「智能体」编辑页改为 IDENTITY/SOUL 两栏；既有智能体行为不变（身份锚为空时注入与此前逐字一致）。
- 新增「远程接入」（微信 ClawBot）：设置页「远程接入」**扫码绑定微信**（iLink login 取码+轮询，confirmed 自动获取并保存 bot_token，无需手填），即可在微信里与本地 agent 完整双向对话——派任务、收节流回复、用编号回复批准风险操作 / 回答 ask_user / 批准计划，agent 与工具仍全程在本地执行（getUpdates 长轮询纯出站，无需公网 relay）。绑定后第一个给 bot 发消息的人自动认作 owner 入白名单；仅白名单 peer 可驱动。新增 SQLite `remote_allowlist` / `remote_bindings` / `remote_channels` 三表与远程管理/配对命令。Telegram / 钉钉 / 媒体消息为后续切片。
- 技能模块改造为文件型：每个技能是 `{HOME}/.siliconagent/skills/<name>/SKILL.md`（YAML frontmatter）；技能页改为内置/用户安装双 Tab，支持拖拽/选 zip/选目录安装、启停、卸载，详情抽屉可渲染 SKILL.md 并预览目录文件。内置技能随 app 内嵌、启动物化、默认启用。移除 app 内新建/编辑技能能力。
- 新增「产物」：Agent 用 add_artifact 登记最终交付文件（绑定产生它的消息），侧栏「产物」面板与会话 feed 内联展示，点击在右侧 Drawer 预览（markdown 渲染 / 纯文本 / 二进制系统打开）。
- 新增 workspace 工作目录：composer 下拉可为会话选择/替换工作目录（随时可改）+ 最近使用的目录，未选时默认 `{home}/.siliconagent/{session_id}`，Agent 文件工具按会话解析沙箱根；任务监控面板展示当前目录名并可用系统文件管理器打开。
- 设置页新增「用量分析」：按日期/模型/会话/小时统计 Token 用量，含缓存命中/写入与命中率，支持 全部/30天/7天 范围（概览/模型/缓存三视图）。
- 新增 SQLite `token_usage` 表，引擎每次模型调用后采集 token 用量（含缓存）。

### Added
- 新增「智能体」：由一个专家**播种**（软复制其指令）的常驻智能体，有自己可编辑的人设、**私有长期记忆**（跨会话记得你），点「对话」即进入绑定该智能体的会话。技能引用其来源专家（删该专家会令技能脱钩，引用完整性校验后续补）。

### Changed
- 角色模板「智能体」正名为「专家」（expert）：界面、命令与数据表统一改名（DB 表 `agents`→`experts`、列 `agent_id`→`expert_id`，启动自动迁移），为后续引入真正自主的「智能体」（带身份与记忆的运行实例）腾出名称。运行态派发/子运行（dispatch/subagent）语义不变。

### Fixed
- 会话运行后台化：引擎 run 跑在独立线程，刷新/重开 app 不再中断正在进行的输出（detached std::thread，维持阻塞+线程、不上 tokio）。
- 刷新/重开期间重复提交被运行锁拦截，不再重复用户消息、不再并发交错。

### Changed
- ask_user 支持一次多问题：每题含主题/单选或多选/干净问题文本/选项，前端分页作答（可跳过），答案结构化回灌模型。schema 由 {question,options} 迁移为 {questions:[…]}（不兼容旧格式）。
- 命令（submit_user_input / 续跑类）改为立即返回；运行态与暂停态由 agent_stream_event 事件（含新增 run_started / run_finished）+ 会话 is_running 驱动，前端刷新后据此重连。
- Provider 流式请求开启 `stream_options.include_usage` 并解析末尾 usage chunk，使流式对话也能采集 token 用量。
